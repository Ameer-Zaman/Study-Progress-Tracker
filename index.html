Q1. The Nullish Fallback Trap

Predict the value of currentSpeed printed to the console:

const speed = 0;
const currentSpeed = speed ?? 100;
console.log(currentSpeed);
✔️ Correct Answer: 0
Professional Explanation: The nullish coalescing operator (??) strictly checks for null or undefined. Since 0 is a valid falsy number, the fallback value of 100 is completely ignored, correctly preserving the 0.

Q2. Closure Scope & Value Increment

Analyze this code. What sequence of logs will execute when calling these functions?

function createCounter(start) {
  let count = start;
  return function() {
    count++;
    console.log(count);
  };
}
const myCounter = createCounter(10);
myCounter();
myCounter();
✔️ Correct Answer: 11, then 12
Professional Explanation: The inner function forms a persistent closure over the variable count. The post-increment operator (count++) completes the increment operation inside local storage before the value is printed to the terminal console on the next line by console.log.
Impossible / Interview

Q3. Object Reference as Map Keys

What is printed by this lookup?

const database = new Map();
database.set([101], "Admin Profile");
console.log(database.get([101]));
✔️ Correct Answer: undefined
Professional Explanation: Arrays are reference types stored in Heap memory. The array literal [101] passed to .set() has a completely different memory address than the array literal [101] passed to .get(). Because Map key matching uses strict equality (===) on reference pointers, the lookup fails.
