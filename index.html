Q1. The Nullish Fallback Trap

Predict the exact value of currentSpeed logged to the console:

const speed = 0;
const currentSpeed = speed ?? 100;
console.log(currentSpeed);
0

Why: The nullish coalescing operator (??) strictly checks for null or undefined. Since 0 is a valid falsy number, the fallback 100 is ignored.

Q2. Closure Scope & Value Increment

What sequence of numbers prints when executing these functions?

function createCounter(start) {
  let count = start;
  return function() {
    count++;
    console.log(count);
  };
}
const myCounter = createCounter(10);
myCounter();
myCounter();
11, then 12

Why: The inner function forms a persistent closure over count. The post-increment operator (count++) completes the addition before the value is printed on the next line.

Q3. Object Reference as Map Keys

What does this map lookup return?

const database = new Map();
database.set([101], "Admin Profile");
console.log(database.get([101]));
undefined

Why: Arrays are reference types in the heap. The array literal [101] in .set() has a different memory address than the [101] passed to .get(), so the lookup fails.
